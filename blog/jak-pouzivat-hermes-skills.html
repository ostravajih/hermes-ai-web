Jak používat dovednosti (Skills)

8. července 2026 · Doba čtení: 6 minut

Dovednosti (Skills) jsou jednou z nejsilnějších funkcí Hermes Agent. Jsou to znovupoužitelné znalostní dokumenty, které agent načítá podle potřeby. Když Hermes vyřeší složitý problém, může si vytvořit skill, který použije příště — bez opakování stejných chyb.

Jak dovednosti fungují

Dovednosti používají progresivní zveřejňování pro úsporu tokenů:

  • Úroveň 0: Agent vidí jen název a popis (~3k tokenů pro desítky dovedností)
  • Úroveň 1: Plný obsah dovednosti se načte, až když ji agent potřebuje
  • Úroveň 2: Konkrétní referenční soubory (skripty, šablony)

Slash příkazy pro dovednosti

# Spuštění konkrétní dovednosti /github-pr-workflow vytvoř PR pro auth refactor # Skládání více dovedností (až 5 najednou) /plan /github-pr-workflow navrhni a implementuj issue #42 # Vyhledání dovednosti /skills search docker

Vytvoření nové dovednosti pomocí /learn

Nejrychlejší způsob — agent sám shromáždí materiál a vytvoří strukturovanou dovednost:

# Z lokálního adresáře /learn REST API v ~/projects/api-docs # Z online dokumentace /learn https://docs.example.com/api # Z konverzace /learn jak jsem právě nastavil CI/CD pipeline

Formát SKILL.md

Každá dovednost je složka s povinným souborem SKILL.md:

--- name: moje-dovednost description: Co dělá version: 1.0.0 --- # Postup 1. Krok jedna 2. Krok dva

Skills Hub — komunitní dovednosti

Hermes podporuje instalaci dovedností z více zdrojů:

  • agentskills.io — oficiální otevřený standard (používaný Claude Code, Codex, Cursor a dalšími)
  • ClawHub — komunitní repozitář
  • GitHub repozitáře — vlastní kolekce
$ hermes skills install docker-deploy $ hermes skills list
💡 Tip: Dovednosti jsou kompatibilní se standardem agentskills.io — můžete je sdílet mezi Hermes, Claude Code, Codex a dalšími agenty!

Dovednosti jsou klíčem k tomu, aby se váš agent sám zlepšoval v průběhu času. Čím déle běží, tím víc dovedností nasbírá — a tím chytřejší je.
