Hermes Agent cron plánovač

10. července 2026 · 7 min

Hermes Agent má vestavěný cron plánovač — váš agent může spouštět úkoly automaticky 24/7, doručovat výsledky na Telegram/Discord/Email, a to vše bez vaší přítomnosti.

Jak cron funguje

Cron plánovač běží uvnitř Hermes Gateway jako samostatný scheduler. Každý úkol má vlastní:

  • Plán — cron výraz, interval, nebo jednorázové spuštění
  • Prompt — instrukce pro agenta (nebo no_agent=true pro script-only úkoly)
  • Doručení — kam poslat výsledek (Telegram, Discord, email, atd.)
  • Skill — volitelně přednačtené dovednosti
  • Script — volitelně skript, který se spustí před agentem (sběr dat)

Vytvoření cron úkolu

# Každé ráno v 8:00 — ranní briefing /cron create "0 8 * * *" "Dej mi ranní briefing: počasí, GitHub notifikace, dnešní úkoly" # Každé 2 hodiny — kontrola PR /cron create "every 2h" "Zkontroluj nové pull requesty a issues na GitHubu" # Každý pátek v 17:00 — týdenní report /cron create "0 17 * * 5" "Vytvoř týdenní report aktivity a pošli ho na email" # Jednorázově — 1. srpna 2026 /cron create "2026-08-01T09:00:00" "Pošli připomenutí: zaplatit fakturu"

Formát cron výrazu

PoziceVýznamHodnoty
1Minuta0-59
2Hodina0-23
3Den v měsíci1-31
4Měsíc1-12
5Den v týdnu0-7 (0=neděle)

Reálné příklady použití

1. Ranní briefing na Telegram

# Každý všední den v 7:30 /cron create "30 7 * * 1-5" "Dej mi ranní přehled: - Dnešní počasí v Praze - Nepřečtené GitHub notifikace - Nové emaily za posledních 24h - Dnešní úkoly z Todoist"

2. Watchdog nad serverem

# Každých 30 minut /cron create "*/30 * * * *" "Zkontroluj: - Využití CPU a RAM (alert pokud > 80%) - Místo na disku (alert pokud < 10 GB) - Zda běží všechny služby (nginx, postgres) - Poslední chyby v /var/log/syslog Pokud je problém, pošli urgentní zprávu."

3. Automatické PR review

# Každou hodinu v pracovní době /cron create "0 9-17 * * 1-5" "Zkontroluj otevřené PR v repozitáři. Pro každé PR: - Zkontroluj diff na problémy - Spusť testy pokud je to možné - Napiš review komentář"

Správa úkolů

$ hermes cron list # Seznam všech úkolů $ hermes cron pause ID # Pozastavení $ hermes cron resume ID # Obnovení $ hermes cron run ID # Okamžité spuštění $ hermes cron remove ID # Odstranění $ hermes cron status # Stav plánovače

Script-only úkoly (no_agent)

Pro úkoly, které nepotřebují AI (např. monitoring, heartbeat), použijte no_agent=true — skript se spustí přímo a jeho stdout se doručí jako zpráva. Pokud je stdout prázdný, zpráva se neodešle (ideální pro watchdog — upozorní jen když je problém).

# CLI: $ hermes cron add --name "Watchdog" --schedule "*/5 * * * *" --script scripts/check_server.sh --no-agent # /cron v chatu: /cron create "*/5 * * * *" script=scripts/check_server.sh no_agent=true

Doručování výsledků

Výsledky cron úkolů lze doručovat na různé platformy:

  • Telegram: deliver: telegram
  • Discord: deliver: discord:KANAL_ID
  • Email: deliver: email
  • Původní chat: deliver: origin (výchozí)
⏰ Cron úkoly běží i když je Hermes vypnutý — scheduler je součástí Gateway, která běží jako systemd služba 24/7. Nastavte si ranní briefing a už nikdy nezapomenete na důležitý úkol.
