Hermes Agent v Dockeru

8. července 2026 · Doba čtení: 4 minuty

Docker je ideální způsob, jak provozovat Hermes Agent izolovaně — bez ovlivnění zbytku systému. Hodí se pro testování, produkční nasazení i vývoj.

Výhody Dockeru pro Hermes

  • ✅ Izolace — agent nemůže ovlivnit hostitelský systém
  • ✅ Reprodukovatelnost — stejné prostředí všude
  • ✅ Jednoduchá správa — jeden příkaz pro start/stop
  • ✅ Perzistentní data — skills, paměť a konfigurace přežijí restart

Instalace

# Instalace Dockeru (Ubuntu) $ sudo apt install docker.io docker-compose -y $ sudo systemctl enable --now docker

docker-compose.yml

version: '3.8' services: hermes: image: hermes-agent:latest volumes: - ./hermes-data:/root/.hermes environment: - OPENAI_API_KEY=sk-... restart: unless-stopped
🔒 Bezpečnost: Nikdy nedávejte API klíče přímo do docker-compose.yml. Použijte .env soubor nebo Docker secrets.

Spuštění

$ docker-compose up -d $ docker-compose logs -f

Agent běží izolovaně v kontejneru, data zůstávají ve složce hermes-data/ na hostiteli.
